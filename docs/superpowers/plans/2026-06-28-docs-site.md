# track_gen Documentation Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a high-quality Sphinx (reStructuredText) documentation site for `track_gen` — public-API reference from enriched docstrings, tuning guides, tutorials, and per-generator deep-dives — that builds cleanly today and is ready to publish to GitHub Pages later.

**Architecture:** A Sphinx project rooted at `docs/` with the Furo theme and autodoc/napoleon. Work proceeds in four phases: a sequential **foundation** (scaffold that builds clean), **docstring enrichment + autodoc pages**, a **parallel fan-out** of independent content pages (each touches only its own file, so parallel subagents never collide), and a sequential **integration** pass that wires the nav, deletes migrated sources, updates the README, and runs the final strict build.

**Tech Stack:** Sphinx, Furo, `sphinx.ext.autodoc`/`napoleon`/`autosummary`/`intersphinx`/`viewcode`/`mathjax`, `sphinx-copybutton`, `sphinx-design`. Existing stack: Python ≥ 3.10, NVIDIA Warp, numpy, matplotlib (dev).

## Global Constraints

- **Authoring format is pure reStructuredText.** No `myst-parser`, no Markdown pages in the build.
- **Sphinx project root is `docs/`.** `conf.py` at `docs/conf.py`; `docs/_build/` is git-ignored; `exclude_patterns` includes `superpowers/**` and `_build` so internal process docs never build.
- **BUILD GATE (every page/config task ends with this, expecting a clean build):**
  `.venv/bin/python -m sphinx -W --keep-going -b html docs docs/_build/html`
  Warnings are errors. A task is not done until this is clean. Run `.venv/bin/pip install -e ".[docs]"` once after Task 1.
- **LINKCHECK (integration only):** `.venv/bin/python -m sphinx -b linkcheck docs docs/_build/linkcheck`. Internal links must pass; known-flaky external URLs may be allow-listed in `conf.py` `linkcheck_ignore`.
- **`:orphan:` strategy:** every content/reference page created in Phases 2–3 starts with `:orphan:` as its very first line. This lets a page build clean under `-W` before it is wired into any toctree, so independent pages land in parallel with no shared-file edits. Phase 4 (Task 20) removes `:orphan:` as it wires the toctrees. While a page is `:orphan:`, it must NOT use `:doc:`/`:ref:` cross-references to sibling pages (those are added in Task 20); use plain text or external links until then.
- **Markdown→rST conversion recipe** (for migration tasks): headings use under/overline adornments (`=` H1, `-` H2, `~` H3, `^` H4); fenced code → `.. code-block:: <lang>`; pipe tables → `.. list-table::` (or grid tables); images → `.. figure:: ../assets/<file>.png` with a caption; `**Note**`/`**Warning**` callouts → `.. note::`/`.. warning::`; inline code → double-backtick literals; math → `:math:\`…\`` inline and `.. math::` blocks. Preserve the prose; do not rewrite surveys beyond conversion + light cleanup.
- **Docstring edits are text-only.** No signature, default-value, or behavior changes. Preserve existing warnings (aliasing, CUDA graph, fixed-batch). NumPy/Google-style sections (`Attributes:`, `Parameters:`, `Returns:`, `Raises:`) only.
- **Figures are deterministic.** Rendering code runs on the Warp `cpu` device with fixed seeds, imports only the public `track_gen` API + matplotlib/numpy/warp, and must NOT import from `viz.param_explorer`. Match the existing `viz/render_readme_assets.py` style.
- **Running the pytest suite requires** `PYTEST_DISABLE_PLUGIN_AUTOLOAD=1` (broken ROS pytest plugins leak from `/opt/ros/humble` and break collection otherwise).
- **Existing `docs/assets/*.png` stay in place** (README image links depend on them). rST pages reference them by relative path.

```python
def render_generator_panels(output_dir: Path = OUT_DIR) -> list[Path]:
    output_dir.mkdir(parents=True, exist_ok=True)
    wp.init()
    written: list[Path] = []
    for idx, (name, label) in enumerate(GENERATORS):
        phase1, valid, chosen = _generate(name, seed=100 + 17 * idx, needed=5)
        ncol = max(1, len(chosen))
        fig, axes = plt.subplots(1, ncol, figsize=(2.3 * ncol, 2.6), dpi=170, facecolor="white")
        axes = axes if ncol > 1 else [axes]
        for ax, env_id in zip(axes, chosen):
            _draw_phase1_output(ax, phase1, valid, env_id, lw=1.5)
        fig.suptitle(f"{label} — representative phase-1 outputs", fontsize=13,
                     fontweight="bold", color="#111827")
        fig.tight_layout(rect=(0, 0, 1, 0.92), w_pad=0.4)
        path = output_dir / f"generator-{name}.png"
        fig.savefig(path, bbox_inches="tight", facecolor="white")
        plt.close(fig)
        written.append(path)
    return written
```

### Task 21: CI workflow (deploy-ready, activation deferred)

**Files:** Create `.github/workflows/docs.yml`

**Interfaces:** Consumes the buildable site. Produces a workflow that builds on PRs and is ready to publish.

- [ ] **Step 1: Create the workflow.**

```yaml
name: docs
on:
  push:
    branches: [main]
  pull_request:
permissions:
  contents: read
  pages: write
  id-token: write
concurrency:
  group: docs-${{ github.ref }}
  cancel-in-progress: true
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - run: pip install -e ".[docs]"
      - run: python -m sphinx -W --keep-going -b html docs docs/_build/html
      - uses: actions/upload-pages-artifact@v3
        with:
          path: docs/_build/html
  deploy:
    needs: build
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```
